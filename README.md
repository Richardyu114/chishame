# 吃啥么（微信小程序 MVP）

一个帮打工人快速决定 **“今天吃什么”** 的微信小程序。

> 核心：先解决“吃什么内容”（饭 / 面 / 蛋白 / 蔬菜），店铺由用户自己选择。

## 产品定位（v0.6）
- 产品名：吃啥么
- 场景：工作日快速决策，不纠结
- 目标：低成本、少功能、界面简洁但有记忆点
- 非目标：不做外卖平台式的附近找店/比价/下单链路

## 当前实现
- 首页卡片（扑克牌式滑动）
  - 今日吃法组合：主食 + 蛋白 + 蔬菜 + 补充
  - 卡片背景图（本地图片资源）
  - 估算热量 + 推荐理由
  - 古人饮食语录（便于分享）
- 随机与决策
  - `今天吃这个`
  - `交给口福`
  - `再翻一页`
  - `换个风味`
- 个性化设置
  - 默认口味（随机/清淡/均衡/下饭/浓香/辛辣/轻食）
  - 忌口标签（海鲜/牛肉/豆制品/鸡蛋/生冷/辛辣）
- 结果页
  - 展示本次吃法结构
  - 支持复制分享文案
  - 底部提供朋友圈分享动作：
    - 生成分享图片（保存到相册后发朋友圈）
    - 直接分享到朋友圈（通过右上角分享菜单）

## v0.4 数据源升级（核心变化）
- 推荐池优先走网络源：
  - 菜谱来源：TheMealDB `random.php`
  - 古诗文语录来源：今日诗词 `all.json`
- 本地结构化池（`foodData.js`）作为兜底：
  - 网络不可用 / 域名未放行 / 返回数据不足时自动回退
  - 回退对用户无感知，不阻断推荐流程
- 口味偏好与忌口仍保留：
  - 对网络菜谱做标签映射（清淡/下饭/浓香/辛辣/轻食）
  - 对忌口标签做关键词过滤（海鲜/牛肉/豆制品/鸡蛋/生冷/辛辣）

## v0.5 视觉与交互升级（Phase 1~3）
- Phase 1（视觉基线）
  - 全局设计 token 收敛：暖白底 + 松柏绿主色 + 雾金点缀
  - 统一卡片/按钮/标签/文字层级（减少页面风格漂移）
- Phase 2（页面落地）
  - 推荐页：顶部信息卡 + 卡片层次重排 + 文案语气统一（如“再翻一页”）
  - 结果页：结构信息改为双列信息排布，强调“今日定案”
  - 设置/引导页：风格统一到同一套中性色与间距系统
- Phase 3（手感与性能）
  - 推荐页异步加载加入竞争保护 token（避免快速切换错位）
  - 轻触反馈（`wx.vibrateShort`）与按钮按压反馈统一
  - 远程图片轻量预热（前两张）提升滑动观感稳定性

## 技术说明
- 前端本地可运行，网络源按可用性自动增强
- 数据存储：`wx.setStorageSync`
- 随机引擎：网络优先 + 本地 fallback
- 图片：本地静态资源（`miniprogram/assets/food`）+ 远程图轻量预热

## 微信小程序域名说明（重要）
若希望在真机/正式环境使用网络源，请在小程序后台配置 request 合法域名：
- `https://www.themealdb.com`
- `https://v1.jinrishici.com`

开发阶段可在微信开发者工具里临时关闭“校验合法域名”进行调试；未配置时会自动回退到本地推荐池。

## 朋友圈分享说明
- 小程序支持在页面中准备 `onShareTimeline` 内容，并引导用户通过右上角菜单完成“分享到朋友圈”。
- 本项目额外提供“生成分享图片”能力，适合发布图文朋友圈：
  1) 点击结果页底部“分享到朋友圈”
  2) 选择“生成分享图片”
  3) 保存到系统相册后发布到朋友圈

## 本地运行（微信开发者工具）
1. 打开微信开发者工具。
2. 导入项目目录：`/home/chishame`。
3. 使用测试 AppID / touristappid 运行。
4. 进入首页后可直接体验“今天吃什么”卡片流。
